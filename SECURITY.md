# Security Policy

## Supported Versions

This project is pre-1.0 and moves quickly. Security fixes are applied on the default branch and included in the next release cut.

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected vulnerabilities.

Report security issues privately to the maintainers with:

- a clear description of the issue
- impact assessment
- reproduction steps or a proof of concept
- affected files, harnesses, or configuration paths

If private vulnerability reporting is enabled on the repository, use that channel. Otherwise, contact the maintainers directly before public disclosure.

We will acknowledge receipt, assess severity, and coordinate a fix and disclosure timeline.
